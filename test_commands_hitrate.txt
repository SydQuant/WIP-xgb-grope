# Key Difference from standard test_commands.txt:
# - All commands include --q_metric "hit_rate" to use hit rate as the quality metric for model selection
# - Log labels include "_hitrate" suffix to differentiate from sharpe-based runs
# - Model selection will prioritize directional accuracy over risk-adjusted returns

# Additional options available:
# --rolling N               : Use rolling window of N days instead of expanding window
# --no_feature_selection    : Skip feature selection (use all features)
# --max_features N          : Limit features to N (default: automatic)
# --n_bootstraps N          : Number of bootstrap samples for p-values (default: 100)
# --inner_val_frac F        : Inner validation fraction (default: 0.2)
# --ewma_alpha A            : EWMA alpha for Q-score (default: 0.1)
# --quality_halflife H      : Quality tracker halflife (default: 63)
# --top_n_models N          : Top N models to select (default: 10% of n_models)
# --reselection_frequency N : Model reselection frequency in folds (default: 1)

# Notes:
# - Hit rate Q-metric focuses on directional accuracy rather than Sharpe ratio
# - Model selection will prefer models with higher hit rates
# - GPU acceleration is automatically detected and used when available
# - Results are saved to xgb_compare/results/ directory
# - Logs are saved to xgb_compare/results/logs/ directory
